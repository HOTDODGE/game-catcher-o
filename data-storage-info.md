# 데이터 저장 및 관리 방식 안내 (Data Storage Info)

본 프로젝트는 별도의 전용 데이터베이스(DB) 서버 없이 **브라우저 로컬 저장소**와 **외부 API 서비스**를 활용하여 사용자 기능을 구현하고 있습니다.

---

## 1. 위시리스트 (Wishlist)
- **저장 위치**: 방문 중인 브라우저의 `localStorage` (`gamecatcher_wishlist` 키 사용)
- **방식**: 
    - 사용자가 하트 아이콘을 클릭하면 게임 정보(ID, 제목, 썸네일 등)를 JSON 형태로 브라우저 내부 저장소에 저장합니다.
- **특징**: 
    - 서버 비용이 발생하지 않으며 속도가 매우 빠릅니다.
    - 단, 기기 간 데이터 동기화가 되지 않으므로 현재 사용 중인 브라우저에서만 목록이 유지됩니다.

## 2. 목표 가격 알림 (Price Alerts)
가격 알림 기능은 로컬 UI 관리와 실제 알림 발송을 위해 두 가지 경로를 동시에 사용합니다.

### A. 로컬 UI 관리 (LocalStorage)
- **저장 위치**: `localStorage` (`user_alerts` 키 사용)
- **목적**: 사용자가 '내 알림 목록'에서 본인이 설정한 알림 정보를 확인하고 삭제할 수 있도록 화면 표시용 데이터를 보관합니다.

### B. 서버 알림 서비스 (CheapShark API)
- **저장 위치**: **CheapShark API 서버**
- **동작**: 
    - 알림 설정 시 사용자의 **이메일**과 **목표 가격**이 CheapShark 서버로 전송됩니다.
    - 실제 게임 가격이 목표가 이하로 떨어지면, CheapShark의 백엔드 시스템에서 자동으로 사용자에게 알림 이메일을 발송합니다.
    - 이 방식 덕분에 서버를 직접 운영하지 않고도 강력한 알림 기능을 제공할 수 있습니다.

---

## 요약 (Summary)

| 기능 | 저장소 | 주요 특징 |
| :--- | :--- | :--- |
| **위시리스트** | 로컬 (LocalStorage) | 기기별 독립 저장, 빠른 응답성 |
| **알림 목록** | 로컬 (LocalStorage) | 사용자 화면 표시 및 관리용 |
| **이메일 알림** | 외부 API (CheapShark) | 실제 가격 변동 감지 및 이메일 발송 담당 |

> [!NOTE]
> 브라우저의 쿠키나 방문 기록을 삭제할 경우 로컬에 저장된 위시리스트와 알림 목록이 초기화될 수 있으니 주의하시기 바랍니다.
