# 서비스 운영 시 고려 사항 및 잠재적 이슈 리포트

GameCatcher 서비스를 실제 도메인에 연결하여 정식 운영할 때 발생할 수 있는 주요 기술적, 정책적 이슈들을 정리한 리포트입니다.

---

## 1. 기술적 이슈 (Technical)

### 🔒 CORS (Cross-Origin Resource Sharing)
- **현상**: 브라우저 보안 정책상, 도메인이 달라지면 외부 API(Steam, CheapShark 등) 서버에서 요청을 차단할 수 있습니다.
- **해결책**: 자체 백엔드 서버(Proxy)를 구성하여 통신하거나, CORS 허용 설정이 된 API 게이트웨이를 사용해야 합니다.

### 🔌 API 호출 제한 (Rate Limiting)
- **현상**: 다수의 사용자가 동시에 접속하여 API를 직접 호출하면 IP 단위로 차단되거나 속도가 저하될 수 있습니다.
- **해결책**: 서버 사이드 캐싱을 도입하여 동일한 데이터에 대한 반복 요청을 줄여야 합니다.

### 🛡️ 보안 노출
- **현상**: 현재는 모든 로직이 클라이언트 사이드(JS)에서 실행되므로, 추후 유료 API 키를 사용할 경우 키가 대중에게 노출될 수 있습니다.
- **해결책**: 환경 변수(Environment Variables)를 사용하여 서버 측에서만 API 키를 관리해야 합니다.

### 🔍 SEO (검색 엔진 최적화)
- **현상**: 클라이언트 사이드 렌더링(CSR) 방식은 검색 엔진 로봇이 내용을 수집하는 데 한계가 있을 수 있습니다.
- **해결책**: Next.js 같은 프레임워크를 사용한 SSR(서버 사이드 렌더링) 또는 SSG(정적 사이트 생성) 도입이 유리합니다.

---

## 2. 운영 및 데이터 이슈 (Operational)

### ⏳ 데이터 업데이트 지연
- **현상**: 제3자 API 서비스의 업데이트 주기에 따라 실제 판매 사이트와 가격 정보에 시차가 발생할 수 있습니다.
- **해결책**: 데이터 소스의 최종 업데이트 시간을 명시하거나, 실시간 확인 버튼을 제공하여 사용자 혼란을 최소화해야 합니다.

### 💾 영구 저장소의 부재
- **현상**: 현재 LocalStorage를 사용하므로 기기 변경 시 위시리스트 등의 데이터가 유실됩니다.
- **해결책**: 회원가입 시스템과 데이터베이스(MySQL, MongoDB 등)를 구축하여 데이터를 중앙 관리해야 합니다.

---

## 3. 법적 및 정책적 이슈 (Legal / Policy)

### 📜 API 사용 약관 (Terms of Service)
- **현상**: CheapShark, Steam 등의 API는 상업적 이용 시 별도의 승인이 필요하거나 제약이 있을 수 있습니다.
- **해결책**: 각 API 제공사의 정책(Commercial Use 등)을 사전에 철저히 검토해야 합니다.

### 🖼️ 저작권 및 상표권
- **현상**: 게임 이미지, 로고 등의 무단 사용은 저작권 분쟁의 소지가 있습니다.
- **해결책**: 각 플랫폼의 제휴 프로그램(Affiliate Program)에 가입하거나, 공식 API 가이드를 준수하여 리소스를 사용해야 합니다.
